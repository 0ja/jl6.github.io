Security Rule: No ports are open on the local router. The Raspberry Pi worker initiates outward polling long-connections to GitHub's secure messaging queue socket automatically.

Step 1: Raspberry Pi Runner Setup

  • Raspberry Pi Terminal
    1. Create Execution Space & Extract Runner Packages

    Run this configuration layout inside your home folder directory:

    mkdir actions-runner && cd actions-runner

# Pull official ARM64 background runner source
curl -o actions-runner-linux-arm64-2.317.0.tar.gz -L https://github.com/actions/runner/releases/download/v2.317.0/actions-runner-linux-arm64-2.317.0.tar.gz

# Unpack binaries
tar xzf ./actions-runner-linux-arm64-2.317.0.tar.gz
  • GitHub UI ➔ Pi Terminal
    2. Register Machine Signature Mapping

    Navigate to Settings ➔ Actions ➔ Runners on GitHub, click New Self-Hosted Runner, and run your customized token sequence link:

    ./config.sh --url https://github.com/jl6/glep --token YOUR_RUNNER_REGISTRATION_TOKEN
  • Raspberry Pi Terminal
    3. Establish Permanent System Service Daemon

    Install the process hook into systemd scripts so it executes automatically when the Pi boots up:

    sudo ./svc.sh install
sudo ./svc.sh start

Phase 2: MacBook Repository Rules

  • MacBook Workspace
    1. Generate Hidden Folder Trees

    Inside the root execution path of your local glep bot repo directory, create the target automation mapping folder:

    mkdir -p .github/workflows
  • MacBook Workspace
    2. Write the Script Blueprint (deploy.yml)

    Create a file named deploy.yml exactly inside that hidden folder chain and paste this automation script block:

    name: Glep Automated Deployment

on:
  push:
    branches: [ main ]

jobs:
  deploy:
    runs-on: self-hosted

    steps:
      - name: Checkout Code Base
        uses: actions/checkout@v4

      - name: Clear and Install Production Dependencies
        run: |
          npm ci --only=production

      - name: Hot-Reload PM2 Process
        run: |
          pm2 restart glep --update-env || pm2 start index.js --name "glep"

The Daily Production Loop

From this point forward, your workflow strictly consists of your local MacBook terminal execution paths. Drop your SSH software entirely.

# 1. Stage any modified tracking files
git add .

# 2. Commit the changes to your local system timeline
git commit -m "Updated scan module logic definitions"

# 3. Fire the pipeline (Pi self-reloads in background automatically)
git push origin main
